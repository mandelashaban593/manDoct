Ok, to get things started you could read some of the info on how this works on the post I made in my blog here: Django AJAX Powered chat Part 2 @ PythonHaven, or visit the wiki section of the project's homepage (whenever it is updated)...

The following links expose the two ways that django-jchat can be used in your site.

  1. The first mode (AKA "standalone" mode) the chat room is instanced and exists by itself, unattached to anything else in your project. This can be useful if you want to create a "global chat" application for your site, or simply chat rooms that exist by themselves.
  2. The second mode (AKA "the power of generic relations" mode) the chat room is binded to an instance of any model in your site, allowing you to create chat rooms for forum topics, user groups, user-user conversations, etc.

Standalone

This example leads you to a page where the chat room instance exists by itself, it is not bound to any other models in the project.

Complex

For this example the database has been populated with dummy instances of a "document" model. The use case I'm covering with this example is that of a site like Google Docs, where you can edit documents and chat with the people editing the document at the same time (it's the only example I could think of, sorry). So, by binding the chatroom instances to each document I can have a separate chat room for each document so that contributors can collaborate in a more Web 2.0'esque way :D.
